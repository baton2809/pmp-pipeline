# SPEC: excel-parser

> Первый скилл pipeline pmp-vs-jira. Читает `Бэклог и цели.xlsx`, лист `Q2_26_оценки_new_name`, извлекает план задач квартала и сохраняет в `.cache/enriched.json`.
>
> Никаких вызовов MCP. Только Excel. Это база для следующих скиллов pipeline.

## 1. Контекст и место в pipeline

```
[Бэклог и цели.xlsx]
        │
        ▼
   excel-parser ◄── (этот скилл)
        │
        ▼
   .cache/enriched.json
        │
        ▼
   jira-enricher → timing-analyzer → report-builder
```

## 2. Цели

- Прочитать жёстко зафиксированный файл `Бэклог и цели.xlsx`
- Найти жёстко зафиксированный лист `Q2_26_оценки_new_name`
- Извлечь 28 задач квартала с CR-ключами, названиями, инициативами, планом А/Р/Т
- Создать структуру `enriched.json` согласно CONTRACT.md, секция "После excel-parser"
- Сохранить в `.cache/enriched.json` рабочей директории

## 3. Анти-цели

- **НЕ** ходить в Jira (это делает следующий скилл)
- **НЕ** генерировать markdown-отчёт (это делает `report-builder`)
- **НЕ** создавать Python-проект — только `helper.py` если необходим
- **НЕ** работать с другими листами/файлами кроме зафиксированных

## 4. Вход и выход

### Вход

- Файл `Бэклог и цели.xlsx` в текущей рабочей директории (жёстко зафиксировано)
- Лист `Q2_26_оценки_new_name` (жёстко зафиксировано)

Если файл отсутствует — сообщить пользователю с точным именем файла, не пытаться найти альтернативы. Если лист отсутствует — то же самое.

### Выход

`.cache/enriched.json` в текущей рабочей директории. Структура — см. CONTRACT.md секция "После excel-parser".

Папку `.cache/` создать если не существует.

## 5. Структура Excel — жёстко зафиксированные колонки

Колонки **по позиции** (не по имени заголовка):

| Колонка | Что | Как обрабатываем |
|---------|-----|------------------|
| **B** | CR (ключ Jira или URL) | парсим regex, strip пробелов |
| **C** | Заказчик | строка как есть |
| **F** | Инициатива | строка, очистить от `\n` |
| **G** | Задача (название) | строка, очистить от `\n`, `\r`, `\t` |
| **J** | Аналитика, первая версия | число или null |
| **K** | Разработка, первая версия | число или null |
| **L** | Тестирование, первая версия | число или null |
| **R** | Аналитика, вторая версия | число или null |
| **S** | Разработка, вторая версия | число или null |
| **T** | Тестирование, вторая версия | число или null |

Строка 1 — заголовки (пропускаем). Данные начинаются со строки 2 или 3 (проверить).

### Выбор версии плана

Для каждой задачи:
1. Если **R, S, T** все три заполнены — берём вторую версию, `source_version = "v2"`
2. Если хотя бы одно из R/S/T заполнено а первая версия пустая — `source_version = "v2"`, неvilled → null
3. Если только **J, K, L** заполнены — берём первую, `source_version = "v1"`
4. Если ни одно — `plan = {analytics: null, development: null, testing: null, total: null, source_version: "none"}`. Задача всё равно попадает в `tasks` (она реально есть в плане Excel), просто без оценок.

### Парсинг CR-ключа

Из значения колонки B:
1. `value.strip()` — убрать пробелы по краям
2. Если пусто или None → задача без CR, попадает в `metadata.skipped_rows` с указанием номера строки
3. Применить regex `r'(ASFC|CRSIGMA|OCRED|TIBDS|ASFS)-\d+'`
4. Взять `match.group(0)` как ключ
5. Если regex не нашёл — `skipped_rows` с пометкой "некорректный формат CR"

## 6. Steps

### Step 1. Проверить файл

Проверить что `Бэклог и цели.xlsx` существует в текущей директории. Если нет — сообщение пользователю:

> Файл `Бэклог и цели.xlsx` не найден в текущей директории. Убедитесь что вы запустили GigaCode в правильной папке.

Завершить.

### Step 2. Открыть лист

Открыть лист `Q2_26_оценки_new_name`. Если листа нет — перечислить доступные листы, попросить пользователя проверить. Завершить.

Реализация — через `helper.py` функция `open_sheet()` использующая `openpyxl.load_workbook(..., data_only=True)`.

### Step 3. Найти строку начала данных

Заголовки в строке 1. Данные начинаются обычно со строки 3 (строка 2 может быть подзаголовком). Логика:
- Идти со строки 2
- Найти первую строку где **в колонке B есть значение** содержащее regex `(ASFC|CRSIGMA|OCRED|TIBDS|ASFS)-\d+`
- Это строка начала данных

### Step 4. Пройти по строкам, извлечь задачи

Для каждой строки от начала данных до `ws.max_row`:
1. Извлечь и распарсить CR-ключ (см. правила выше)
2. Если skip — добавить в `metadata.skipped_rows`, продолжить
3. Извлечь название, инициативу, заказчика
4. Извлечь план обеих версий, выбрать актуальную
5. Создать объект `task` согласно CONTRACT.md
6. Добавить в `tasks` массив

### Step 5. Очистить тексты

Для `task_name` и `initiative`:
- Заменить `\n`, `\r`, `\t` на пробел
- Свернуть множественные пробелы в один
- Strip по краям

Это делается в `helper.py` функция `clean_text(value)`.

### Step 6. Сохранить json

1. Создать папку `.cache/` если её нет
2. Сформировать объект `enriched` с metadata + tasks + пустыми `jira: null` и `timing: null` для каждой задачи + пустой `epics: []`
3. Записать в `.cache/enriched.json` с `indent=2, ensure_ascii=False`

### Step 7. Сообщить пользователю

В чат вывести краткую сводку:
- Обработан файл: `Бэклог и цели.xlsx`, лист `Q2_26_оценки_new_name`
- Извлечено задач: N
- Пропущено строк (без CR): M
- Создан файл: `.cache/enriched.json`
- Следующий шаг: запустите `jira-enricher`

- `main.py`, `process.py`, `run_*.py`, `generate_*.py`
- `__pycache__/`, `requirements.txt`, `pyproject.toml`, виртуальные окружения
- Любые `.py` файлы кроме одного `helper.py`
- Запуск скилла как Python-приложения — SKILL.md остаётся главной точкой входа, `helper.py` это набор функций

## 8. КРИТИЧНО: формат SKILL.md

SKILL.md — это **инструкция для агента**, а не Python-скрипт. Каждый Step описан на естественном языке. Когда нужны вычисления — агент либо пишет inline `python3 -c '...'` через bash, либо импортирует функцию из `helper.py`.

Пример **правильного** шага:
```
Шаг 2. Открыть Excel-файл.

Импортировать `helper.py` (он рядом со SKILL.md в скиллах) и вызвать:
  workbook = helper.open_workbook("Бэклог и цели.xlsx")
  sheet = helper.get_sheet(workbook, "Q2_26_оценки_new_name")

Если функция вернула ошибку — вывести сообщение пользователю и завершить.
```

**Запрещено** в SKILL.md:
- Полные Python-программы внутри инструкции
- `# TODO`, `# здесь будет код`, заглушки
- Описания вроде "напишите скрипт который..." — скилл сам описывает что делать

## 9. Guardrails

- READ-ONLY для Excel: только читаем, не модифицируем
- Никаких MCP-вызовов
- Только один python-файл: `helper.py`
- Жёстко зафиксированы имя файла и лист
- Не создаём `report.md` — это работа другого скилла
- Не интерпретируем данные — только извлекаем

## 10. Edge cases

| Ситуация | Поведение |
|----------|-----------|
| Файл `Бэклог и цели.xlsx` не найден | Сообщить точное имя, завершить |
| Лист `Q2_26_оценки_new_name` не найден | Перечислить доступные листы, завершить |
| В колонке B нет ни одной строки с CR-ключом | Сообщить "не найдены CR-ключи в колонке B", завершить |
| Строка с пустой колонкой B | Пропустить, не добавлять в skipped_rows |
| Строка с мусором в колонке B (например "TBD" или "?") | skipped_rows с пометкой "не удалось распознать CR" |
| Дубликат CR-ключа в плане (как `CRSIGMA-22127` в Q2) | Обе строки добавляются как отдельные задачи. В `metadata` отметка "дубликаты ключей: [...]" |
| Колонки R/S/T пустые но J/K/L заполнены | Используем v1 версию плана |
| Все версии плана пустые | `plan` с null полями, задача всё равно в tasks |
| Excel содержит несколько листов | Используем только Q2_26_оценки_new_name, остальные игнорируем |

## 11. Антипаттерны

### Критические

- **Создание лишних .py файлов** (`main.py`, `process.py`, ...) — в прошлой v3 GigaCode создал 4 файла и пайплайн поломался. Только `helper.py`.
- **Заглушки в SKILL.md** (`# здесь будет парсинг`) — SKILL.md должен быть готов к работе.
- **Поиск по имени заголовка** вместо позиции колонки — нестабильно, имена могут меняться.

### Обычные

- Записывать в `enriched.json` сырой ответ openpyxl без очистки
- Падать на одной плохой строке вместо skipped_rows
- Не создавать `.cache/` если её нет
- Перезаписывать существующий `enriched.json` без сохранения предыдущих секций (для idempotency перезаписываем целиком — это первый скилл, ничего сохранять не надо)

## 12. Критерий успеха

После запуска:
1. Файл `.cache/enriched.json` создан и валиден (структура совпадает с CONTRACT.md)
2. Количество задач в массиве `tasks` соответствует количеству CR-ключей в Excel
3. У каждой задачи заполнены `cr_key`, `task_name`, `initiative`, `plan`
4. `metadata.skills_completed = ["excel-parser"]`
5. В чате выведена сводка
6. Никаких лишних файлов в рабочей директории
