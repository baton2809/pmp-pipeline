# report-builder — четвёртый (последний) скилл pipeline

Генерирует финальный markdown-отчёт из накопленного `.cache/enriched.json`. Не ходит в Jira, не модифицирует данные — только формирует текст.

## Место в pipeline

```
1. excel-parser
2. jira-enricher
3. timing-analyzer (опционально)
4. report-builder    ← вы здесь    → report.md
```

## Условный рендеринг

Скилл работает в двух режимах в зависимости от того что в `.cache/enriched.json`:

**Полный режим** (если timing-analyzer отработал):
- Все 7 секций отчёта
- Колонки "Факт А/Р/Т"
- Секция "Застрявшие"

**Без timing режим** (если timing-analyzer не запускался):
- 6 секций (без "Застрявшие")
- Без колонок "Факт А/Р/Т"
- В дисклеймере явная пометка "для расчёта факта запустите timing-analyzer"

Это сделано чтобы можно было быстро увидеть отчёт без changelog-анализа, и потом добавить если нужно.

## Подготовка

MCP не нужен — этот скилл не ходит в Jira.

Должны отработать как минимум `excel-parser` и `jira-enricher`. `timing-analyzer` опционален.

## Запуск

### Шаг 1. Сгенерировать

```bash
cd report-builder
gigacode
```

Вставить промпт из `PROMPT.md`. Получить `skill/report-builder/SKILL.md` + `helper.py`.

Перед установкой проверить:
- 2 файла в `skill/report-builder/`
- helper.py содержит функции: `truncate_words`, `compute_marker`, `compute_summary`, `format_main_table`, `format_epics_section`, `format_teams_section`, `format_stuck_section`, `build_report`
- SKILL.md не делает MCP-вызовов
- SKILL.md не модифицирует `enriched.json`
- Условный рендеринг (`if timing_available`) реализован

### Шаг 2. Установить

```bash
cp -r skill/report-builder ~/.gigacode/skills/
```

### Шаг 3. Запустить

```bash
gigacode
```

В чате: "запусти report-builder".

После прогона:
- `report.md` создан в рабочей директории
- В чате сводка с указанием размера и секций

## Что показать Наталье

Открыть `report.md` в редакторе с поддержкой markdown (VS Code, Obsidian, Telegram, GitHub). Главные секции для разговора:

1. **Дисклеймер** ("Что в отчёте и чего нет") — сразу проговорить ограничения
2. **Секция 5 "Застрявшие"** (если есть) — конкретный список для разбора с командой
3. **Секция 3 "Срез по эпикам"** — обзор где сколько работы
4. **Секция 4 "Срез по командам"** — кто чем загружен

Спросить:
- Это полезный взгляд на портфель?
- Что добавить/убрать?
- Какие пороги "застрявших" лучше: × 2 от плана или другой?
- Хочется ли v4 (контроль дат ИФТ/ПСИ/ПРОМ)?

## Сколько занимает

~2-5 секунд. Никаких внешних вызовов, только локальная обработка json и формирование текста.

## Повторный запуск

Можно запускать сколько угодно раз — отчёт перезаписывается. Если в `enriched.json` ничего не изменилось — отчёт будет идентичным.

Если хочется обновить отчёт с новыми данными — сначала перезапустить `jira-enricher` (и опционально `timing-analyzer`), потом `report-builder`.
