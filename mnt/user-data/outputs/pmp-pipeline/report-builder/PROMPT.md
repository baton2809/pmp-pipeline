# PROMPT для GigaCode CLI — report-builder

```
Прочитай SPEC.md и CONTRACT.md в корне репозитория. report-builder — четвёртый (последний) скилл pipeline pmp-vs-jira.

Сгенерируй два файла:

1. skill/report-builder/SKILL.md
2. skill/report-builder/helper.py

Что делает скилл:
- Читает .cache/enriched.json
- Валидирует что excel-parser и jira-enricher отработали (timing-analyzer опционален)
- Генерирует markdown-отчёт из 7 секций
- Сохраняет в report.md в рабочей директории
- НЕ ходит в Jira, НЕ читает Excel, НЕ модифицирует enriched.json

Критически важно:

1. Раздел 5 SPEC.md — точная структура отчёта из 7 секций с дисклеймером. Соблюдать порядок.

2. Раздел 6 SPEC.md — правила форматирования: нормализация текста (убрать \n, обрезка по словам через …), лимиты по колонкам, формат маркеров, формат эпика "<ключ> <имя_до_25_симв>".

3. Раздел 6.5 — все 10 маркеров с условиями. Реализовать функцию compute_marker в helper.py.

4. Условный рендеринг: если timing-analyzer не отработал — отчёт работает без колонок Факт А/Р/Т и без секции 5 (Застрявшие), с явной пометкой в дисклеймере.

5. Раздел 8 SPEC.md (файлы): ТОЛЬКО helper.py. Никаких main.py, process.py, run_*.py, generate_*.py, __pycache__.

6. НЕ модифицировать .cache/enriched.json — этот скилл write-only для report.md.

7. НЕ делать MCP-вызовов — все данные уже в enriched.json.

8. UTF-8 без BOM. Кириллица сохраняется как есть, не как \uXXXX.

9. Эпик: формат "<ключ> <имя>" (имя обрезано до 25 символов). Если имя null — только ключ. Если эпика нет — "—".

10. Команда: формат "<значение>" или "(по assignee) <имя>" с явным указанием источника. Если ничего — "—".

11. План А/Р/Т: "23/23/15" или "—" если оценок нет.

12. Факт А/Р/Т (если есть timing): "30/240/0" целые числа или "—".

Когда сгенерируешь — выведи в чат:
- Файлы которые создал
- Функции в helper.py
- Один пример строки таблицы (как будет выглядеть)
- Подтверди что обрезка идёт по словам (не посередине)
- Подтверди что отчёт работает БЕЗ timing-analyzer (условный рендеринг)
- Подтверди что нет MCP-вызовов и нет модификации enriched.json
```
